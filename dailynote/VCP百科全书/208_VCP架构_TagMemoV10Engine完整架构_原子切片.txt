## 核心结论
TagMemoV10Engine 是 VCP 记忆系统的 V10 Alpha 实验引擎完整实现。它集成了不可变快照、溯源视图、Artifact仓库、双条件算子、双尺度场求解、候选超集、路径几何、相对拓扑匹配、DSTC观测、直接锚点读出、派生资产管理等16个子模块。V10 Alpha 是实验版本，不替换 V9.2 生产搜索，RiverMemo Topology V3 已接管生产执行路径。

## 工作机制
1. **引擎构造** (`TagMemoV10Engine.js:54`):
   - `constructor(db, tagIndex, config, ragParams, options)`
   - 持有 `v9Engine` 引用（V9降级兼容）
   - `_activeArtifact` / `_artifactGeneration` 管理活动 Artifact
   - `_conditionedOperatorCache`（大小限制1，命中统计 hits/misses）
   - ArtifactRepository 和 DerivedAssetManager 可注入或自动创建

2. **模块依赖链** (16个导入):
   - `immutable.js` → hashStable, immutableSnapshot, 只读Map/Set/CSR视图
   - `provenance.js` → buildProvenanceView, 溯源快照
   - `riverMemoArtifactRepository.js` → Artifact持久化仓库
   - `agentConditioner.js` → createDualConditionedOperators（双条件算子）
   - `scaledFieldSolver.js` → solveDualScaledFields（双尺度场求解）
   - `candidateSuperset.js` → buildCandidateSuperset（候选超集构建）
   - `curveProjector.js` → projectCandidateCurves（候选曲线投影）
   - `unifiedPathGeometry.js` → evaluateCandidateCurves（路径几何评估）
   - `relativeTopologyMatcher.js` → evaluateRelativeTopologyBatch（相对拓扑匹配）
   - `dstcObservables.js` → computeDstcBatch（DSTC观测计算）
   - `directAnchorReadout.js` → computeDirectAnchorBatch（直接锚点读出）
   - `experimentArms.js` → scoreExperimentArm, runExperimentArms（实验臂）
   - `fieldWorkspace.js` → createFieldWorkspace（场工作区）
   - `derivedAssetManager.js` → V10DerivedAssetManager（派生资产管理）
   - `riverObservability.js` → computeRiverObservability（可观测性计算）

3. **RiverMemo 的传播域**:
   RiverMemo 的传播域是全局 Tag 图，不是任一 Diary/Chunk ANN 索引。每个活动 Artifact 只常驻一份全局编译算子。
