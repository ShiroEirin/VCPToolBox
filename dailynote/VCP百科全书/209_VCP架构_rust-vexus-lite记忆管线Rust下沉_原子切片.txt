## 核心结论
rust-vexus-lite 是 VCP 记忆系统的 Rust 原生计算底座。本次上游更新新增了4个核心Rust源文件，将 TagMemo V9.1/V9.2 的 Spike 感知、EPA/Pyramid/Spike 融合管线、DSTC 双轨语义一致性以及 RiverMemo Topology V3 的全部重计算下沉至 Rust，使用 rayon 并行计算框架。JS 层只负责查询场准备和结果组装，不再执行任何重量级传播计算。

## 四大Rust模块

### 1. memo_sensing.rs — V9 Spike 感知观测
- ALGORITHM_VERSION: `tagmemo.spike-v9.1-rust-shared`
- OBSERVATION_SCHEMA: `vexus-unified-memo-observation-v1`
- 功能: 将 V9 的 Spike 传播（激活波在Tag图上的传播）实现为Rust原生
- 关键结构:
  - `SenseInput` — 包含 queryId, seeds[], config
  - `SenseSeed` — id, energy(默认1.0), sourceType(默认"seed")
  - `SenseConfig` — max_safe_hops, base_momentum, firing_threshold, base_decay, wormhole_decay, tension_threshold, max_neighbors_per_node, return_flow_factor, fir_gamma
  - `SenseOutput` — 观测结果
  - `ObservationNode` / `ObservationParent` / `ObservationEdge` — 河网结构
  - `SenseDiagnostics` — 诊断信息
- 导出函数: `sense_typed()`, `sense()`

### 2. memo_pipeline.rs — V9.2 EPA/Pyramid/Spike 融合管线
- PIPELINE_ALGORITHM: `tagmemo.epa-pyramid-spike-fusion-v9.2-rust`
- PIPELINE_SCHEMA: `vexus-unified-memo-pipeline-v1`
- 功能: 将 V9.2 的 EPA（能量传播）、Pyramid（层级金字塔）、Spike（尖峰传播）三种降噪源融合为统一管线
- 依赖: `usearch::Index`（向量索引）, `rusqlite`（数据库）, `base64`
- 内部函数: `positive()`, `clamp01()`, `magnitude()`, `dot()`, `cosine()`, `normalize()`
- 导出: `MemoPipelineTask`, `run_with_runtime`

### 3. memo_dtsc.rs — DSTC 双轨语义一致性
- ALGORITHM_VERSION: `tagmemo.dtsc-v9.2.1-rust-js-parity`
- RESULT_SCHEMA: `vexus-dtsc-native-result-v1`
- 功能: Dual-Track Semantic Consistency 双轨语义一致性计算，保证 Rust 和 JS 实现的输出一致
- 使用 rayon 并行计算
- 依赖: `rivermemo_topology_v3::{load_artifact_from_runtime, MemoRuntime, NativeArtifact}`
- 导出: `MemoDtscTask`, `rerank_with_runtime`

### 4. rivermemo_topology_v3.rs — RiverMemo Topology V3 原生内核
- ALGORITHM_VERSION: `rivermemo.topology-v3.1-rust`
- RESULT_SCHEMA: `rivermemo-topology-v3-native-result-v1`
- 功能: RiverMemo Topology V3 完整计算内核（详见切片207）
- 使用 rayon 并行、rusqlite 数据库、flate2 Gzip 解压、serde_json
- 导出: `NativeArtifact`, `MemoRuntime`, `rerank_rivermemo_topology_v3`
