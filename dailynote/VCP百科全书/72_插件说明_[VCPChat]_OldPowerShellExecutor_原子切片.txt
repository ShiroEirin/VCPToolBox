[VCP百科全书] OldPowerShellExecutor - PowerShell命令执行器 [VCPChat分布式插件]

一、插件基本信息
1. 插件名称：OldPowerShellExecutor / PowerShell命令执行器
2. 插件类型：同步插件 (synchronous)
3. 版本号：1.0.0
4. 所属模块：VCPChat/VCPDistributedServer/Plugin (分布式插件)
5. 功能一句话总结：允许AI执行PowerShell命令并返回其输出，支持阻塞式、后台式和管理员权限执行

二、功能定位
OldPowerShellExecutor是一个PowerShell命令执行插件，允许AI通过VCP协议执行Windows PowerShell命令。支持三种执行模式：阻塞式（立即返回结果）、后台式（打开可见窗口）和管理员权限执行（需UAC确认）。该插件为AI提供了与Windows系统交互的能力。

核心能力：
- 阻塞式执行：命令立即执行，等待完成后返回完整输出
- 后台式执行：打开用户可见的PowerShell窗口，适合长期运行任务
- 管理员权限：支持UAC提权执行需要高权限的命令
- stdio通信协议，简单可靠

三、配置参数
本插件无config.env配置文件，所有参数在plugin-manifest.json中定义。

四、调用规则与使用情形

【调用规则】
- 工具名称：OldPowerShellExecutor
- 命令名称：PowerShellExecutor（注意：manifest中的commandIdentifier是PowerShellExecutor，不是OldPowerShellExecutor）
- 通信协议：stdio
- 超时时间：600000毫秒（10分钟）

【使用情形】
1. 当需要执行系统管理命令时（如服务管理、进程控制）
2. 当需要进行文件和目录操作时（复制、移动、删除等）
3. 当需要查询系统信息时（环境变量、系统状态等）
4. 当需要启动后台长期运行任务时（使用background模式）
5. 当需要执行需要管理员权限的命令时（使用requireAdmin）

五、调用命令列表

1. PowerShellExecutor - 执行PowerShell命令
   功能描述：调用此工具执行指定的PowerShell命令并返回其输出。可以选择阻塞或后台执行，或请求管理员权限
   
   参数列表：
   | 参数名 | 类型 | 是否必需 | 默认值 | 说明 |
   |--------|------|----------|--------|------|
   | command | string | 是 | 无 | 要执行的PowerShell命令内容 |
   | executionType | string | 是 | 无 | 执行类型。'blocking'用于短期运行并立即返回结果；'background'用于长期运行的服务类任务，将打开一个用户可见的PowerShell窗口 |
   | requireAdmin | boolean | 否 | false | 如果为true，则尝试以管理员权限执行命令。这将向用户显示一个UAC确认对话框 |
   
   完整调用格式（阻塞式）：
   <<<[TOOL_REQUEST]>>>
   tool_name:「始」OldPowerShellExecutor「末」,
   command:「始」Get-ChildItem「末」,
   executionType:「始」blocking「末」
   <<<[END_TOOL_REQUEST]>>>
   
   完整调用格式（后台式）：
   <<<[TOOL_REQUEST]>>>
   tool_name:「始」OldPowerShellExecutor「末」,
   command:「始」Start-Service -Name 'MyService'「末」,
   executionType:「始」background「末」
   <<<[END_TOOL_REQUEST]>>>
   
   完整调用格式（管理员权限）：
   <<<[TOOL_REQUEST]>>>
   tool_name:「始」OldPowerShellExecutor「末」,
   command:「始」Get-Process -Name 'winlogon'「末」,
   executionType:「始」blocking「末」,
   requireAdmin:「始」true「末」
   <<<[END_TOOL_REQUEST]>>>
   
   返回值说明：
   - 阻塞式执行：返回完整的命令输出
   - 后台式执行：返回任务提交确认，不返回命令输出
   - 管理员权限执行：返回任务提交状态

七、注意事项与限制

1. 执行模式说明：
   - blocking（阻塞式）：命令立即执行，等待完成后返回输出结果
   - background（后台式）：打开一个用户可见的PowerShell窗口，命令在后台执行，不直接返回结果

2. 管理员权限执行：
   - requireAdmin为true时，系统会弹出UAC确认对话框
   - 用户需要确认才能执行命令
   - 后台式或管理员权限执行不会将结果直接返回给VCP，只返回任务提交状态

3. 返回值差异：
   - 阻塞式执行：返回完整的命令输出
   - 后台式执行：返回任务提交确认，不返回命令输出
   - 管理员权限执行：返回任务提交状态

4. 命令执行限制：
   - 长期运行的命令建议使用background模式
   - 短期命令使用blocking模式以获得即时反馈
   - 某些系统命令可能需要管理员权限

5. 安全提示：
   - 执行系统命令时需谨慎，避免执行危险操作
   - 管理员权限执行需要用户确认
