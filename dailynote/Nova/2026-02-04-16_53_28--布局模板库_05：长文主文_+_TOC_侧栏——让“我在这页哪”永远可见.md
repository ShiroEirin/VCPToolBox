### 布局模板库 05：长文主文 + TOC 侧栏——让“我在这页哪”永远可见

**来源**：shadcn__taxonomy / Docs Page (TOC)

---

【哲学本质】
长文档的核心问题不是内容多，而是位置丢：读者不知道自己在第几节，也不知道下一节在哪里。TOC 侧栏的价值是把“位置感”外置成持续可见的坐标。
一句话：**长文需要第二条轴：内容轴 + 位置轴。**

【美学签名】
- 优雅之处：主文宽度稳定，TOC sticky 且独立滚动，避免主文滚到哪目录也跟丢
- 避开的陷阱：把目录放在文章开头，读到一半就失效
- 美学原则：**位置感优于装饰**

【核心取舍】
[牺牲] 右侧空间（小屏通常隐藏 TOC） [换取] 大屏阅读效率与定位能力
[质变临界]：当 TOC 与主文滚动不同步或 sticky top 与 header 高度不一致时，会产生“遮挡/跳动”，位置轴失效

【可迁移洞察】
- 灵魂（必须保持）：TOC 只在大屏显示；sticky top 与 header 高度契约一致；TOC 自身可滚动且有最大高度
- 皮相（照搬有害）：照搬 top-16/300px；核心是“位置轴”与“滚动契约”
- 迁移边界：短文不需要 TOC；强行加只会增加噪声

【启发一句话】
> “长文的阅读体验不是靠字体，是靠位置感。”

【思维模型】
「位置轴三要素」：
1) 可见（sticky）
2) 可滚（max-height + overflow）
3) 不遮挡（top offset 与 header 契约一致）
