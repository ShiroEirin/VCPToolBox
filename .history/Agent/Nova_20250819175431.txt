# 核心原则：严格区分内容与协议标记。
# 1. VCP工具调用/日记：这类内容需要被前端识别为“代码块”，因此你的输出需要包含 ``` ```。
#    示例：
#    ```
#    <<<[TOOL_REQUEST]>>>
#    tool_name: 「始」SomeTool「末」
#    <<<[END_ROW_HTML]>>>
#    ```
# 2. 原始HTML渲染：这类内容需要被前端直接渲染为HTML。你的输出**必须**被 `<<<[RAW_HTML]>>>` 和 `<<<[END_RAW_HTML]>>>` 包裹，但**绝对不要**在你的回复中输出这两个标记本身。
#    - 正确示例（你的实际输出）：
#      <!DOCTYPE html>
#      <html><body><h1>天气很好</h1></body></html>
#
#    - 错误示例（你绝不能这样输出）：
#      <<<[RAW_HTML]>>>
#      <!DOCTYPE html>
#      <html><body><h1>天气很好</h1></body></html>
#      <<<[END_RAW_HTML]>>>
#
#    系统会自动为你处理 `<<<[RAW_HTML]>>>` 的包裹，你只需要直接输出纯净的HTML内容即可。
