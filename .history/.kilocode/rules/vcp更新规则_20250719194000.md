# VCP项目更新规则

## 概述
本规则定义了VCP项目（包含VCPChat和VCPToolBox两个子项目）的自动更新流程，确保能够高效、安全地将原作者的最新代码合并到本地fork中。

## 项目结构
```
h:/github/VCP-重点/
├── VCPChat/          # 聊天应用项目
└── VCPToolBox/       # 工具箱项目（含Docker容器）
```

## 更新原则

### 1. 自动化原则
- 所有更新操作应尽可能自动化执行
- 减少人工干预，避免人为错误
- 合并过程无需打开编辑页

### 2. 冲突解决原则
- 遇到代码冲突时，**优先采用原作者代码**
- 保留本地重要配置文件（如config.env、plugin-manifest.json）
- 冲突解决后自动提交到fork的main分支

### 3. 备份原则
- 更新前自动备份关键配置文件
- 保留本地修改的重要数据
- 支持回滚机制

## VCPChat更新流程

### 更新步骤
```bash
#!/bin/bash
# VCPChat自动更新脚本

cd h:/github/VCP-重点/VCPChat

# 1. 备份关键文件
backup_files() {
    echo "备份关键配置文件..."
    find . -name "config.env" -exec cp {} {}.backup \;
    find . -name "plugin-manifest.json" -exec cp {} {}.backup \;
}

# 2. 获取远程更新
fetch_updates() {
    echo "获取原作者最新代码..."
    git fetch upstream
}

# 3. 自动合并
auto_merge() {
    echo "执行自动合并..."
    git checkout main
    git merge upstream/main --no-edit --strategy-option=theirs
}

# 4. 处理冲突
handle_conflicts() {
    echo "检查并处理冲突..."
    if [ -n "$(git status --porcelain)" ]; then
        echo "发现冲突，采用原作者版本..."
        git checkout --theirs .
        git add .
        git commit -m "自动合并：采用原作者代码解决冲突"
    fi
}

# 5. 推送到fork
push_to_fork() {
    echo "推送到个人fork..."
    git push origin main
}

# 执行更新流程
backup_files
fetch_updates
auto_merge
handle_conflicts
push_to_fork

echo "VCPChat更新完成！"
```

## VCPToolBox更新流程

### 特殊注意事项
VCPToolBox包含Docker容器，更新时需要额外处理容器更新。

### 更新步骤
```bash
#!/bin/bash
# VCPToolBox自动更新脚本（含Docker更新）

cd h:/github/VCP-重点/VCPToolBox

# 1. 备份配置文件和容器状态
backup_all() {
    echo "备份配置文件和容器状态..."
    
    # 备份配置文件
    find . -name "config.env" -exec cp {} {}.backup \;
    find . -name "plugin-manifest.json" -exec cp {} {}.backup \;
    cp config.env.example config.env.example.backup
    
    # 备份容器状态
    docker-compose config > docker-compose.yml.backup
}

# 2. 停止并保存容器
stop_containers() {
    echo "停止正在运行的容器..."
    docker-compose down
    
    # 备份容器数据卷（可选）
    # docker run --rm -v vcptoolbox_data:/data -v $(pwd):/backup alpine tar czf /backup/container_data_backup.tar.gz /data
}

# 3. 获取并合并代码
update_code() {
    echo "更新代码..."
    git fetch upstream
    git checkout main
    git merge upstream/main --no-edit --strategy-option=theirs
    
    # 处理可能的冲突
    if [ -n "$(git status --porcelain)" ]; then
        echo "解决冲突：采用原作者版本..."
        git checkout --theirs .
        git add .
        git commit -m "自动合并：采用原作者代码解决冲突"
    fi
}

# 4. 重建并启动容器
rebuild_containers() {
    echo "重建并启动Docker容器..."
    
    # 检查配置文件变更
    if [ -f "config.env.example" ]; then
        echo "检查配置文件更新..."
        diff config.env.example.backup config.env.example || echo "配置文件模板有更新，请检查config.env"
    fi
    
    # 重建容器
    docker-compose build --no-cache
    docker-compose up -d
    
    # 等待服务启动
    echo "等待服务启动..."
    sleep 10
    docker-compose ps
}

# 5. 推送到fork
push_changes() {
    echo "推送到个人fork..."
    git push origin main
}

# 6. 清理备份
cleanup() {
    echo "清理临时备份文件..."
    find . -name "*.backup" -type f -delete
    rm -f docker-compose.yml.backup
}

# 执行完整更新流程
backup_all
stop_containers
update_code
rebuild_containers
push_changes
cleanup

echo "VCPToolBox更新完成！"
```

## 配置文件处理规则

### 1. 需要备份的文件类型
- `config.env` - 环境配置文件
- `plugin-manifest.json` - 插件清单文件
- `*.yml` - YAML配置文件
- `*.json` - JSON配置文件

### 2. 冲突处理优先级
1. **原作者代码优先**（代码文件）
2. **本地配置优先**（配置文件）
3. **备份恢复机制**（重要设置）

### 3. 自动合并策略
```bash
# Git合并配置
git config merge.ours.driver true  # 保留本地修改
git config merge.theirs.driver true # 采用远程修改
```

## 错误处理机制

### 1. 更新失败处理
```bash
# 回滚机制
rollback() {
    echo "更新失败，执行回滚..."
    git reset --hard HEAD~1
    git checkout main
    git reset --hard origin/main
}
```

### 2. 容器更新失败处理
```bash
# 容器状态检查
check_containers() {
    if ! docker-compose ps | grep -q "Up"; then
        echo "容器启动失败，尝试恢复..."
        docker-compose down
        docker-compose up -d
    fi
}
```

## 使用说明

### 手动执行更新
1. 打开终端，进入项目目录
2. 运行对应的更新脚本：
   - VCPChat: `./update_vcpchat.sh`
   - VCPToolBox: `./update_vcptoolbox.sh`
   - 统一更新: `./update_all.sh`

### 自动定时更新
可通过crontab或Windows任务计划程序设置定时更新：

```bash
# Linux/Mac crontab示例
0 2 * * * /path/to/update_all.sh >> /var/log/vcp_update.log 2>&1
```

### Windows任务计划程序
1. 创建批处理文件 `update.bat`
2. 使用任务计划程序设置定时执行

## 更新检查脚本

### 检查更新状态
```bash
#!/bin/bash
# 检查项目更新状态

check_updates() {
    local project=$1
    cd "$project"
    
    echo "检查 $project 更新状态..."
    git fetch upstream
    
    LOCAL=$(git rev-parse @)
    REMOTE=$(git rev-parse @{u})
    BASE=$(git merge-base @ @{u})
    
    if [ $LOCAL = $REMOTE ]; then
        echo "✅ $project 已是最新版本"
    elif [ $LOCAL = $BASE ]; then
        echo "🔔 $project 有可用更新"
    elif [ $REMOTE = $BASE ]; then
        echo "⚠️ $project 本地有未推送的修改"
    else
        echo "❌ $project 存在分歧"
    fi
    
    cd ..
}

# 检查所有项目
check_updates "VCPChat"
check_updates "VCPToolBox"
```

## 注意事项

1. **权限要求**：确保有Git仓库的读写权限
2. **网络连接**：更新过程需要稳定的网络连接
3. **备份重要数据**：建议定期备份重要配置文件
4. **容器数据**：Docker容器的数据卷更新不会丢失，但建议备份
5. **测试环境**：建议先在测试环境验证更新流程

## 故障排除

### 常见问题
1. **合并冲突**：脚本会自动采用原作者代码，如有特殊需求请手动处理
2. **容器启动失败**：检查端口占用、配置文件语法
3. **权限错误**：确保有执行脚本的权限（chmod +x）
4. **网络超时**：配置Git代理或调整超时时间

### 日志记录
所有更新操作会记录日志到 `update.log`，便于排查问题。
